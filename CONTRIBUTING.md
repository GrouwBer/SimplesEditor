# Contributing

Obrigado por contribuir! Siga estes passos:

1. Atualize seu fork e clone o repositório.
2. Crie uma branch a partir de `dev` com o formato `feat/<issue-number>` ou `fix/<issue-number>`.
3. Abra um PR direcionado para `dev` com descrição clara, link para a issue e checklist de critérios de aceite.
4. Solicite revisores e mencione os responsáveis (use @usuario).
5. Mantenha os commits pequenos e atômicos; use mensagens no formato `type(scope): descrição`.
6. Após aprovação e CI verde, merge direcionado para `dev`.

Dicas adicionais:
- Execute os testes locais antes de submeter.
- Atualize a documentação (README, SPRINTS.md) se necessário.
